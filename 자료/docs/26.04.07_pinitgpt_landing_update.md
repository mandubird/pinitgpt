## 수정 1. "30+ early users" → "93 users" 텍스트 변경

### 현재 코드 (찾아야 할 문자열)
```
30+ early users
```

### 변경 후
```
93 early users
```

### 위치
랜딩페이지 중간 섹션 "Built for builders, not enterprises" 아래  
Early usage 통계 카드 안에 있음

### Cursor 지시 프롬프트
```
랜딩페이지에서 "30+ early users" 텍스트를 찾아서 "93 early users"로 변경해줘.
```

## 수정 3. (선택) Free 한계 도달 시 앱 내 업그레이드 유도

> 이건 랜딩이 아니라 익스텐션 코드 수정이므로 별도 작업

### 로직
```
핀 저장 시도 시
→ 현재 핀 수 카운트 확인
→ 10개 이상이면:
   "무료 플랜은 최대 10개까지 저장할 수 있습니다.
    Pro로 업그레이드하면 무제한으로 저장할 수 있어요."
   + "Pro 업그레이드" 버튼 → Gumroad 링크
```

### Cursor 지시 프롬프트
```
크롬 익스텐션 코드에서 핀을 저장하는 함수를 찾아줘.
거기에 핀 개수가 10개 이상일 때 업그레이드 안내 메시지를 띄우는 로직을 추가해줘.
메시지: "Free 플랜은 최대 10개까지 저장됩니다. Pro로 업그레이드하면 무제한으로 사용할 수 있어요."
버튼: "Pro 업그레이드 ($4.99)" → href: [Gumroad 링크]
```

# Pro 기능 검토: 클라우드 동기화 · CSV/Notion 내보내기

## 1. 클라우드 동기화

### 결제할 수밖에 없는 이유
- **"집에서 저장한 Pin을 회사에서도 보고 싶다면?"** → 가장 강력한 유료 전환 요소로 적합.

### 기술적 검토

| 항목 | 내용 |
|------|------|
| **현재 구조** | Pin 데이터는 `localStorage`(pinitgpt_pins)에만 존재. 서버·계정 없음. |
| **필요한 것** | (1) **백엔드** – 데이터를 저장·동기화할 서버 또는 BaaS (Firebase, Supabase 등) (2) **사용자 식별** – 동일 사용자로 인식할 방법 (이메일 로그인, OAuth, 또는 링크 코드 방식) (3) **동기화 로직** – 올리기/받기, 충돌 처리(같은 Pin을 두 기기에서 수정한 경우) |
| **확장 프로그램 변경** | localStorage 읽기/쓰기 → (선택) 로컬 캐시 + 원격 API 호출. 오프라인 시 대기열 후 동기화. |
| **권한/보안** | manifest에 백엔드 URL 호스트 권한 추가. API 키는 확장 내에 포함 또는 사용자 입력. |
| **프라이버시** | 현재 "외부 서버 통신 없음·모든 데이터 로컬" 명시와 충돌. 스토어 설명·개인정보 처리 방침 수정 필요. |

### 결론

- **구현 가능 여부**: ✅ **가능** (백엔드 + 식별자 + 동기화 로직 구현 시).
- **난이도**: 높음. 백엔드 구축, 인증/식별, 충돌 해결, 오프라인 대응 필요.
- **추천**: **Pro 전용 Phase 2 기능**으로 기획하고, 런칭 후 백엔드 설계·구축을 진행한 뒤 도입. 초기 런칭 시에는 "클라우드 동기화 (예정)" 문구로 기대감만 노출 가능.

## 2. CSV / Notion 내보내기

### 가치
- 단순 조회를 넘어, Pin을 **노션이나 엑셀에서 한 번에 활용**하게 해서 헤비 유저 전환·유지에 유리.

### 2-1. CSV 내보내기

| 항목 | 내용 |
|------|------|
| **데이터 소스** | 이미 `getPins()`로 전체 Pin 조회 가능. |
| **구현** | 버튼 클릭 → Pin 배열 → CSV 문자열 생성(헤더: label, **chatTitle**, text, tags, createdAt) → `Blob` + `URL.createObjectURL` + `<a download="pinitgpt-pins-….csv">` 로 다운로드. chatTitle은 좌측 사이드바 채팅 링크 텍스트. tags는 `parseTags()`로 문자열/배열 호환. |
| **서버/로그인** | 불필요. 완전 클라이언트 처리. |
| **위치** | popup 또는 사이드바 하단 "CSV 내보내기" 버튼. |

**결론**: ✅ **즉시 추가 가능.** 작업량 적음.

---

### 2-2. Notion 내보내기

**방식 두 가지:**

| 방식 | 설명 | 난이도 | 서버 |
|------|------|--------|------|
| **A. Notion API** | 사용자 Notion 연동 토큰 입력 → API로 페이지/DB 생성 후 Pin 삽입 | 중 | 불필요(클라이언트에서 Notion API 직접 호출) |
| **B. 파일 내보내기** | Notion이 임포트할 수 있는 **마크다운** 또는 **CSV**로 내보내기 → 사용자가 Notion에서 "Import"로 업로드 | 낮음 | 불필요 |

**A (Notion API)**  
- Notion 개발자 등록, Integration 생성, 사용자가 "연동 토큰"을 확장에 입력해야 함.  
- manifest에 `https://api.notion.com` 호스트 권한 추가.  
- 구현 가능하나, 사용자 설정 단계가 있어 진입 장벽이 있음.

**B (마크다운/CSV)**  
- CSV와 동일한 데이터로 **마크다운 테이블** 또는 **리스트** 문자열 생성 → 파일 다운로드.  
- 사용자는 Notion에서 "Import"로 해당 파일 불러오기.  
- 서버·로그인·Notion API 권한 전부 불필요. **CSV 내보내기와 거의 동일한 난이도.**

**결론**  
- **Notion API 연동**: ✅ 가능. Pro 기능으로 "Notion으로 보내기" 버튼 + 토큰 입력 UI로 추가 가능.  
- **Notion 임포트용 파일(MD/CSV)**: ✅ 바로 가능. CSV와 함께 또는 "Notion용 내보내기"로 마크다운 파일 제공하면 됨.

## 3. 요약 표

| 기능 | 추가 가능 여부 | 조건 / 비고 |
|------|----------------|-------------|
| **클라우드 동기화** | ✅ 가능 | 백엔드 + 사용자 식별 + 동기화 로직. Phase 2·Pro 전용 권장. |
| **CSV 내보내기** | ✅ 즉시 가능 | 서버 없음. getPins() → CSV → 다운로드만 구현하면 됨. |
| **Notion 내보내기** | ✅ 가능 | (1) API: Notion 연동 토큰 + manifest 권한 (2) 파일: 마크다운/CSV로 내보내기 후 Notion Import (서버 없음, 구현 쉬움). |

---

## 4. 구현 우선순위 제안

1. **CSV 내보내기** – 바로 추가 가능, 헤비 유저 만족·전환에 도움.  
2. **Notion용 파일 내보내기** – CSV와 유사한 방식으로 "Notion에서 Import할 수 있는 마크다운" 제공.  
3. **Notion API 직접 연동** – 사용자 설정 필요, Pro 전용으로 나중에 추가.  
4. **클라우드 동기화** – 백엔드 설계·구축 후 Pro 핵심 기능으로 단계적 도입.

이 순서로 가면 "결제할 수밖에 없는 이유"(클라우드)는 로드맵에 넣고, **CSV/Notion 내보내기는 곧바로** 유료 전환 요소로 쓸 수 있음.

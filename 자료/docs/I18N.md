# PinItGPT 다국어(i18n) 구조

UI 텍스트는 하드코딩되지 않고 **번역 키 + JSON** 로 관리됩니다. 기본 언어는 **영어**이며, 브라우저 언어·설정에 따라 자동/수동 전환 가능합니다.

---

## 1. 구조 요약

| 항목 | 설명 |
|------|------|
| **기본 언어** | 영어 (`en`) |
| **자동 감지** | `navigator.language` (접두사 `ko` → `ko`, 그 외 → `en`) |
| **수동 변경** | 사이드바 하단 **언어 선택 UI**에서 선택 시 `pinitgpt_locale` 저장. 또는 직접 `chrome.storage.local.set({ pinitgpt_locale: "en" })` |
| **번역 함수** | `t(key)` — 키에 해당하는 문자열 반환, 없으면 키 그대로 반환 |

---

## 2. 폴더·파일

```
/locales
  en.json   ← 기본 (영어)
  ko.json   ← 한국어
```

- **content script**: `content.js` 상단에서 `loadLocale(cb)` 로 JSON 로드 후 `__pinitgpt_messages` 에 넣고, 모든 UI는 `t('key')` 사용.
- **popup**: `popup.js` 에서 `loadPopupLocale(cb)` 후 `applyPopupI18n()` 으로 `data-i18n` / `data-i18n-html` / `data-i18n-placeholder` 속성 기준으로 텍스트 치환.
- **manifest**: `web_accessible_resources` 에 `locales/*.json` 등록 (content script fetch 용).

---

## 3. 언어 추가 방법

1. `locales/` 아래에 `{lang}.json` 추가 (예: `ja.json`).
2. `content.js` 의 `getLocale()` 에서 `stored === "ja"` 등 허용 언어로 처리.
3. 사이드바 하단 언어 선택 UI에 새 옵션 추가 (드롭다운에 "日本語" 등). 선택 시 `setLocaleAndRefresh(lang)` 와 동일하게 처리.

**새 언어는 JSON 파일만 추가하고, 허용 값·UI 옵션만 확장하면 됩니다.**

---

## 3-2. 언어 선택 UI (사이드바 하단)

- **위치**: 사이드바 맨 하단. `🌐 English` 또는 `🌐 한국어` 표시.
- **동작**: 클릭 시 드롭다운 (English / 한국어) 표시. 옵션 선택 시:
  - `chrome.storage.local` 에 `pinitgpt_locale` 저장
  - 해당 locale JSON 재로드 → `__pinitgpt_messages` 갱신
  - `refreshSidebarI18n()` 으로 사이드바 전체 문구 재적용
  - `renderPins()` 로 리스트·뱃지·게이지 등 재렌더
- **새로고침 없이** 즉시 반영.
- 구현: `content.js` 내 `setLocaleAndRefresh(lang)`, `refreshSidebarI18n()`, `#pinitgpt-lang-trigger` / `#pinitgpt-lang-dropdown`.

---

## 4. 키 규칙

- **로딩**: `loading_navigating`, `loading_long`
- **사이드바**: `plan_free`, `plan_pro`, `pin_gauge`, `search`, `view_*`, `category_*`, `cat_starred` 등
- **모달**: `modal_type_title`, `modal_tag_title`, `modal_save_btn` 등
- **팝업**: `popup_title`, `popup_sub`, `popup_license_enter` 등
- **언어 선택**: `lang_name_en` (English), `lang_name_ko` (한국어)
- **플레이스홀더**: JSON 에 `{0}`, `{1}` 또는 `%s` 사용 시 코드에서 `.replace()` 로 치환.

## 6. Pro+ 개발 시점 언어 확장 (예정)

**Pro+ 버전 개발 시점**에 아래 언어를 추가할 예정입니다.

| 언어 | locale 코드 | 비고 |
|------|-------------|------|
| **일본어** | `ja` | `locales/ja.json` 추가, 드롭다운에「日本語」옵션 |
| **독일어** | `de` | `locales/de.json` 추가, 드롭다운에 "Deutsch" 옵션 |

**구현 순서 (Pro+ Phase)**  
1. `locales/ja.json`, `locales/de.json` 생성 (en.json 키 기준 복사 후 번역)  
2. `content.js` 의 `getLocale()` 에 `"ja"`, `"de"` 허용  
3. 사이드바 언어 선택 드롭다운에 항목 추가  
4. `locales/en.json` 에 `lang_name_ja`, `lang_name_de` 키 추가 (표시명용)  
5. popup 쪽 `applyPopupI18n()` 이 새 locale을 읽을 수 있는지 확인  

기존 **§3 언어 추가 방법**과 동일한 패턴으로 확장하면 됨.
